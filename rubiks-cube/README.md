# 🧩 魔術方塊速解教練（CFOP Solve Coach）

輸入你打亂後的 6 面顏色，系統用速解玩家的 **CFOP 解法**（Cross → F2L → OLL → PLL）一步一步、手把手教你把方塊還原，並附 **3D 動畫**示範每一步怎麼轉（含寬轉、中層、整體旋轉）。全程在你的瀏覽器離線計算，不會把資料傳到任何伺服器。

> 線上使用：部署到 GitHub Pages 後，網址為 `https://<你的帳號>.github.io/shiba-poker/rubiks-cube/`
> 本機使用：直接用瀏覽器打開 `rubiks-cube/index.html` 即可。

---

## CFOP 四大步驟

| 步驟 | 名稱 | 在做什麼 |
|---|---|---|
| 1 | **Cross（十字）** | 在底面把四個白邊排成十字，並對齊四周中心。系統算的是「最短十字」（≤8 步）。|
| 2 | **F2L（前兩層）** | 把白角＋它旁邊的中層邊「配成一對」一次塞進去，一次一組共 4 組，同時完成第一、二層。|
| 3 | **OLL（頂面轉正）** | 用 **57 式** 中的其中一式，一個公式把整個頂面一次翻成黃色。系統會辨識你是哪一式並顯示公式與編號。|
| 4 | **PLL（頂層排列）** | 用 **21 式** 中的其中一式，一個公式把頂層的角和邊一次排到定位，完成還原。系統會顯示是哪一個 perm（T、Y、Ua…）。|

每一步都會顯示：**標準轉法代號**、**白話說明**、以及 **3D 動畫**。可上一步／下一步／播放／調速／直接看完成，也能拖曳方塊轉視角。

## 轉法代號怎麼看

沒符號 = 順時針 90°、加 `'` = 逆時針 90°、加 `2` = 180°（都以面對那一面看）。

| 類型 | 代號 | 意思 |
|---|---|---|
| 單層 | `U D R L F B` | 上／下／右／左／前／後 一層 |
| 兩層（小寫） | `r l u d f b` | 該側的「外兩層」一起轉（如 `r` = 右邊兩層）|
| 中間層 | `M` `E` `S` | `M` 跟左層方向、`E` 跟下層方向、`S` 跟前層方向 |
| 整顆旋轉 | `x` `y` `z` | 整顆方塊翻轉（`x` 跟右層、`y` 跟上層、`z` 跟前層）|

## 建議擺法

為了讓教學裡的顏色名稱和你看到的一致，建議把方塊擺成：
**黃色朝上（U）、白色朝下（D）、綠色朝前（F）**，四周中間列由左到右是 左(橘) 前(綠) 右(紅) 後(藍)。
（配色不同也沒關係，系統會依「中心塊」自動判斷，解法一樣正確。）

## 技術說明

純前端、零外部相依、可離線執行。四個檔案：

| 檔案 | 內容 |
|---|---|
| `js/cube.js` | 方塊幾何引擎：26 個 cubie 的座標與貼紙朝向；支援單層／寬轉（`r`）／中層（`M/E/S`）／整體旋轉（`x/y/z`）；facelet 字串輸入輸出。|
| `js/algs.js` | **OLL 57 式 + PLL 21 式** 標準公式表。|
| `js/solver.js` | 求解器。Cross 與 F2L 用 **IDA\* + 距離表（pattern database）**；OLL/PLL 以「套用公式後驗證」的方式辨識屬於哪一式（保證選到能還原的公式），並自動補上會被省略的整體旋轉。若辨識不到（理論上不會）會退回兩段式後備解，確保一定能還原。|
| `js/app.js` | 介面：展開圖輸入、驗證、3D 立方體渲染與逐步動畫（含寬轉／旋轉）、播放控制與白話說明。|

### 正確性驗證

引擎與求解器在 Node 上以**大量隨機打亂**測試：

- 面轉引擎：`U⁴=I`、`(R U R' U')⁶=I`、`x⁴=I`、`r = R M'`、`x = R M' L'`、打亂＋逆序列＝還原等性質。
- OLL/PLL：每一式都驗證「只改變頂層、保留前兩層」；並確認 **57 式互不重複、21 式互不重複**（完整覆蓋）。
- 端到端：**700 組隨機打亂 100% 還原，且 OLL/PLL 全部走一次公式辨識（0 次退回後備）**。
- 3D 動畫轉向以無頭瀏覽器截圖比對（`F` 順＝右排往下、`U` 順＝前排往左、寬轉 `r` 帶動兩層）。

### 想自己在 Node 跑一次

```bash
cd rubiks-cube/js
node -e '
const E=require("./cube.js"), S=require("./solver.js");
let c=E.solvedCube(); E.applyMoves(c,"R U F2 L D B R2 U D F");
const r=S.solve(E.fromFacelets(E.toFacelets(c)));
console.log("solved:", r.solvedCheck);
r.stages.forEach(s=>console.log(" ", s.title, "->", S.movesToStr(s.moves)));
'
```

## 已知限制

- 平均步數約 55–70 步（CFOP 兩段合一的頂層讓步數比新手法少很多）。
- Cross／F2L 走的是「可靠且短」的解，不是特定命名的 F2L case；OLL／PLL 則是標準的 57／21 式並顯示編號與名稱。
- 顏色以標準六色（白黃紅橙綠藍）為準；非標準配色仍可正確求解，只是教學文字中的顏色名稱可能對不上。
- 第一次計算會先建立距離表，約需 1 秒左右。
