# 🧩 魔術方塊速解教練（Rubik's Cube Solve Coach）

輸入你打亂後的 6 面顏色，系統用**最好懂的「層先法（Layer‑by‑Layer）」**一步一步、手把手教你把方塊還原，並附 **3D 動畫**示範每一步怎麼轉。全程在你的瀏覽器離線計算，不會把資料傳到任何伺服器。

> 線上使用：部署到 GitHub Pages 後，網址為 `https://<你的帳號>.github.io/shiba-poker/rubiks-cube/`
> 本機使用：直接用瀏覽器打開 `rubiks-cube/index.html` 即可。

---

## 這套系統能做什麼

1. **輸入現況**：用一張「展開圖」，對照你手上打亂的方塊，把 54 個小面的顏色點進去。
2. **自動驗證**：檢查顏色數量、中心塊、以及是否為「有可能還原」的方塊，貼錯會提醒你。
3. **產生解法**：用層先法算出完整步驟，並拆成 5 大階段：
   - ① 底部白十字
   - ② 前兩層配對（白角＋中層邊，一組一組放）
   - ③ 頂面黃十字
   - ④ 頂面全變黃
   - ⑤ 對齊完成
4. **手把手教學**：
   - 可 **上一步 / 下一步 / 播放 / 調速 / 直接看完成**。
   - 每一步同時顯示 **標準轉法代號**（U、R'、F2…）與**白話說明**（「把右層順時針轉 90°」）。
   - **3D 立方體**會實際轉給你看，可用手指／滑鼠拖曳轉視角。

## 轉法代號怎麼看

每個代號＝轉「一整層」90°（面對那一面看）：

| 代號 | 意思 | 代號 | 意思 |
|---|---|---|---|
| `U` | 上層 | `D` | 下層 |
| `R` | 右層 | `L` | 左層 |
| `F` | 前層 | `B` | 後層 |

- 沒有符號 = **順時針** 90°
- 加 `'`（如 `R'`）= **逆時針** 90°
- 加 `2`（如 `R2`）= 轉 **180°**

## 建議擺法

為了讓教學裡的顏色名稱和你看到的一致，建議把方塊擺成：
**黃色朝上（U）、白色朝下（D）、綠色朝前（F）**，四周中間列由左到右是 左(橘) 前(綠) 右(紅) 後(藍)。
（如果你的方塊配色不同也沒關係，系統會依「中心塊」自動判斷，解法一樣正確。）

## 技術說明

純前端、零外部相依、可離線執行。三個檔案：

| 檔案 | 內容 |
|---|---|
| `js/cube.js` | 方塊幾何引擎：以 26 個 cubie 的座標與貼紙朝向表示狀態，面轉＝90° 整數旋轉；含 facelet 字串輸入／輸出。|
| `js/solver.js` | 層先法求解器。底十字與前兩層用 **IDA\* 搜尋＋預先算好的距離表（pattern database）** 保證正確且快速；頂層（OLL/PLL）用**巨集 BFS**（以 `F R U R' U' F'`、Sune、A‑perm、U‑perm 等固定公式為單位）涵蓋所有情況。輸出會做基本化簡（合併同面、消去互逆）。|
| `js/app.js` | 介面：展開圖輸入、驗證、3D 立方體渲染與逐步動畫、播放控制與白話說明。|

### 正確性驗證

`solver.js` 的引擎與求解器都在 Node 上以**數千組隨機打亂**測試過：每組打亂後求解，再把解法套回去，確認 100% 回到還原狀態；面轉引擎另以 `U⁴=I`、`(R U R' U')⁶=I`、打亂＋逆序列＝還原等性質驗證。3D 動畫的轉向（順／逆時針）以無頭瀏覽器截圖比對過。

### 想自己在 Node 跑測試

```bash
cd rubiks-cube/js
node -e '
const E=require("./cube.js"), S=require("./solver.js");
let c=E.solvedCube(); E.applyMoves(c,"R U F2 L D B R2 U D F");
const r=S.solve(E.fromFacelets(E.toFacelets(c)));
console.log("solved:", r.solvedCheck, "stages:", r.stages.length);
'
```

## 已知限制

- 採「新手層先法」，步數會比高手的最短解多（平均約 80–95 步），但每一步都好懂、好跟。
- 顏色以標準六色（白黃紅橙綠藍）為準；非標準配色仍可正確求解，只是教學文字中的顏色名稱可能對不上。
- 第一次計算會先建立距離表，約需 1 秒左右。
